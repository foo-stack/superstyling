# Getting Started — Next.js

Zero to `<Button>` on a fresh `create-next-app`. Works with both App Router and Pages Router via subpath imports.

## 1. Install

```bash
yarn add @superstyling/core @superstyling/next @superstyling/icons tamagui
```

Tamagui is a peer dep of `@superstyling/core`; install the matching version (currently `2.0.0-rc.41`).

## 2. Create `tamagui.config.ts`

```tsx
// tamagui.config.ts

export const system = createSystem({
  // your overrides go here; empty input = defaults
});

export default system.config;

declare module "tamagui" {
  interface TamaguiCustomConfig extends typeof system.config {}
}
```

## 3. Wrap `next.config.mjs`

```tsx
// next.config.mjs

export default withSuperStyling({
  config: "./tamagui.config.ts",
})({
  reactStrictMode: true,
});
```

## 4a. App Router — add the `ColorModeScript`

```tsx
// app/layout.tsx

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <head>
        <ColorModeScript initialMode="light" />
      </head>
      <body>
        <SuperStylingProvider>{children}</SuperStylingProvider>
      </body>
    </html>
  );
}
```

## 4b. Pages Router — use the drop-in Document

```tsx
// pages/_document.tsx
export { default } from "@superstyling/next/pages";

// pages/_app.tsx
import type { AppProps } from "next/app";

export default function App({ Component, pageProps }: AppProps) {
  return (
    <SuperStylingProvider>
      <Component {...pageProps} />
    </SuperStylingProvider>
  );
}
```

## 5. Render a Button

```tsx
export default function HomePage() {
  return <Button colorScheme="blue">Hello from Superstyling</Button>;
}
```

## Troubleshooting

:::info[Hydration warnings on html]
The `suppressHydrationWarning` prop on `<html>` is intentional — the `ColorModeScript` mutates the class list before React hydrates. Without it, React complains every load.
:::

:::warning[Module not found: react-native]
You're probably missing a peer. Tamagui aliases `react-native` to `react-native-web` via the Next plugin — make sure the plugin is wrapping your config as shown above.
:::
