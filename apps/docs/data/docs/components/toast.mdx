# Toast — useToast · useNativeToast

<StatusRow tier="1" platforms="Cross-platform" since="v0.2.0" />

Two hooks, same API. Pick the one that matches the vibe you want.

| Hook               | iOS / Android                                 | Web                                  |
| ------------------ | --------------------------------------------- | ------------------------------------ |
| `useToast()`       | Tamagui primitive (cross-platform consistent) | Tamagui primitive                    |
| `useNativeToast()` | `burnt` (native feel)                         | Tamagui primitive (same as useToast) |

## Import

```tsx
import { useToast, useNativeToast } from "@superstyling/core";
```

## Usage

```tsx
function SaveButton() {
  const toast = useToast();
  return (
    <Button
      onPress={async () => {
        await save();
        toast({ title: "Saved", status: "success" });
      }}
    >
      Save
    </Button>
  );
}
```

## Imperative API

```tsx
const id = toast({ title: "Loading…", status: "loading", duration: null });

// Later:
toast.close(id);
toast.closeAll();
```

## When to use which

:::tip[Default: `useToast()`]
Consistent-looking toasts across platforms. Best for cross-platform apps where brand feel > platform feel.
:::

:::info[Native: `useNativeToast()`]
iOS users see iOS-style toasts, Android users see Material-style. Best when your app is a "wrapped native experience" and breaking platform convention would feel off.
:::

`useNativeToast()` silently falls back to the Tamagui implementation if `burnt` isn't installed — so web-only builds never need the dep.
